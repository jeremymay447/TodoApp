## Getting Started

### Prerequisites

- Node.js 18+ installed
- npm or yarn
- **Backend API running** (see todo-app-backend folder)

### Installation

1. Install dependencies:
```bash
npm install
```

2. Create a `.env` file (copy from `.env.example`):
```bash
cp .env.example .env
```

3. Make sure your .NET API is running at `http://localhost:5000`

4. Start the development server:
```bash
npm run dev
```

5. Open your browser to `http://localhost:3000`

### Environment Variables

Create a `.env` file in the root directory:

```env
VITE_API_URL=http://localhost:5000/api
```

For production, update this to your deployed API URL.

### Running with Backend

**Important:** The frontend requires the .NET backend to be running!

1. Start the backend API first:
   ```bash
   cd ../todo-app-backend
   dotnet run
   ```

2. Then start the frontend:
   ```bash
   npm run dev
   ```

If the API is not running, you'll see an error banner with a retry button.

### Build for Production

```bash
npm run build
```

The production-ready files will be in the `dist/` folder.

### Preview Production Build

```bash
npm run preview
```

## How It Works

### State Management
- Uses React's built-in `useState` hook for state
- All todos stored in a single array in App.jsx
- Fetches from .NET API on component mount
- Updates persist to database via API calls

### API Integration
- `todoApi.js` service handles all HTTP requests
- Uses native `fetch` API
- Automatic error handling and loading states
- Base URL configurable via environment variables

### Data Structure

Each todo has:
```javascript
{
  id: number,              // Auto-generated by database
  title: string,
  description: string,
  completed: boolean,
  createdAt: ISO string,
  updatedAt: ISO string
}
```

## API Endpoints Used

- `GET /api/todos` - Fetch all todos
- `POST /api/todos` - Create new todo
- `PUT /api/todos/{id}` - Update todo
- `PATCH /api/todos/{id}/toggle` - Toggle completion
- `DELETE /api/todos/{id}` - Delete todo

## Troubleshooting

### "Failed to load todos" error
- Make sure the .NET backend is running at `http://localhost:5000`
- Check that CORS is properly configured in the backend
- Verify the `VITE_API_URL` in your `.env` file

### CORS errors
- The backend is configured to allow `http://localhost:3000` and `http://localhost:5173`
- If you're using a different port, update the CORS policy in `Program.cs`
